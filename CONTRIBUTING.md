# Contributing

Thanks for taking a look at the project.

This is my first game, and it was built in a very AI-assisted, vibe-coded way. I am still learning, so practical feedback is welcome.

## Good Ways To Help

- Report gameplay bugs or broken rules
- Suggest AI improvements
- Improve UI clarity, feedback, and polish
- Help package and test builds on more platforms
- Improve documentation, screenshots, and release notes

## Before You Open A PR

- Keep changes focused
- Explain what changed and why
- Mention any Unity version or package assumptions
- Include screenshots or short clips for visible UI changes when possible

## Technical Notes

- Unity version used in this repo: `6000.3.8f1`
- Main startup scene: `Assets/ReversiGame/BootStrap/Scene/Bootstrap.unity`
- Current architecture uses Zenject/Extenject and a scene-driven flow

## Collaboration Note

The original core direction I brought into this project was the game-state flow and the AI rules. Everything else has been built as part of an ongoing collaboration and learning process. If you want to contribute, constructive and direct feedback is the most useful kind.
